# Blueprint 002
# User Journey

**Projekt:** Alpine Career

**Version:** 1.0

**Status:** Draft

---

# Ziel

Diese User Journey beschreibt den vollständigen Ablauf eines Nutzers – vom ersten Besuch der Website bis zur erfolgreichen Stellenzusage.

---

# Phase 1 – Landingpage

Der Nutzer besucht alpinecareer.ch.

Ziele:
- Vertrauen schaffen
- Produkt erklären
- Preise zeigen
- Demo anbieten
- Registrierung starten

CTA:
- Kostenlos starten

---

# Phase 2 – Registrierung

Möglichkeiten:
- Google Login
- E-Mail + Passwort

Nach erfolgreicher Registrierung startet automatisch das Onboarding.

---

# Phase 3 – Onboarding

Der Nutzer erhält eine kurze Einführung.

Anschliessend werden eingerichtet:
- Persönliche Daten
- Karriereprofil
- Dokumente
- Google
- Telegram

---

# Phase 4 – Karriereprofil

Erfasst werden:
- Wunschberuf
- Region
- Pensum
- Lohn
- Homeoffice
- Pendelzeit
- Branchen
- No-Gos

---

# Phase 5 – Dokumente

Upload:
- Lebenslauf
- Arbeitszeugnisse
- Diplome
- Zertifikate

Die KI analysiert alle Dokumente.

---

# Phase 6 – Integrationen

Google:
- Gmail
- Drive
- Kalender

Telegram:
- Bot verbinden
- Testnachricht

---

# Phase 7 – Dashboard

Der Nutzer sieht:
- Neue passende Stellen
- Monatsziel
- Offene Bewerbungen
- Interviews
- Erinnerungen
- Match Score
- Confidence Score

---

# Phase 8 – Neue Stelle

Eine passende Stelle wird gefunden.

Die KI:
- bewertet sie
- erstellt Match Score
- bereitet Bewerbung vor

Der Nutzer erhält eine Telegram-Nachricht zur Freigabe.

---

# Phase 9 – Bewerbung

Die KI erstellt:
- E-Mail
- Motivationsschreiben
- passende CV-Version

Der Nutzer prüft und gibt frei.

Erst danach wird versendet.

---

# Phase 10 – Interview

Nach einer Einladung erstellt Alpine Career:
- Kalendereintrag
- Erinnerungen
- Firmenanalyse
- Interviewfragen
- Antwortvorschläge
- Confidence Mode

---

# Phase 11 – Nachbereitung

- Gesprächsnotizen
- Follow-up-Mail
- Status aktualisieren

---

# Phase 12 – Zusage

- Vertrag archivieren
- Bewerbung abschliessen
- Dashboard aktualisieren

---

# Fehlerfälle

- Google nicht verbunden
- Telegram nicht verbunden
- CV fehlt
- Stelle wurde gelöscht
- Bewerbung bereits versendet

---

# Ziel

Der Nutzer soll den gesamten Bewerbungsprozess an einem Ort organisieren können und jederzeit den Überblick behalten.
