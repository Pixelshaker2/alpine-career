# Development Playbook

## Sprint Ablauf

1. Blueprint lesen
2. Anforderungen extrahieren
3. Technischen Plan erstellen
4. Risiken identifizieren
5. Implementieren
6. Tests
7. Review
8. Dokumentation aktualisieren

## Commit Regeln

- Kleine Commits
- Ein Feature pro Branch
- CI muss grün sein

## Review

Vor jedem Merge prüfen:
- Architektur
- Sicherheit
- Performance
- Tests
- Dokumentation
